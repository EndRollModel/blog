mac - 換機的一些紀錄

老樣子 來個事發經過

公司IMac是2017的Intel版本

2023年apple發布macOS14(sonoma)時 宣布部分舊款電腦無法升級sonoma 公司電腦也在列表內 猶如被判了死刑

結果後續發現xcode還是能安心build17 想說應該還會為了後續os更新xcode吧 後續就沒關心這事情了

直到今年ios18發布後 客戶抱怨使用我們的app閃退了 發現公司電腦根本build不到ios 18 手邊也沒任何方法可以測試

公司只好買了一台新的IMAC M3 但因為硬碟大小不同 不能用時光機轉移

只好把專案放到隨身碟移植過去了

然後繞了一大圈把專案環境設定好後 決定做個紀錄 免得到時候相關問題又會把自己卡死

xcode

憑證問題：

xcode登入apple帳號後 另外產生一張新的憑證了 然後apple那邊就還要針對這張憑證去新增

先在舊的電腦上 - xcode設定 - accounts - 找到team的使用者（組織） - manage certificates - 將憑證匯出 - 轉移到新電腦點選憑證使用匯入

之後再到apple developer上將新的電腦自動生成的憑證加入各專案後下載使用即可了

模擬器問題：

因為手上沒有ios18實機 所以需要使用模擬器來run 18的環境

基於build失敗的原因去找了幾篇文章後說是架構問題要排除(arm64)的裝置

最後找到說僅需要開啟使用Rosetta啟動的模擬器即可
(主要原因是因為專案架構上使用cocoapods或是Swift Package Manager
cocoapods不支援arm64架構的內容 所以要排除內容)
但最終找到可以使用rosetta版本的模擬器
Command中輸入 

softwareupdate –install-rosetta

同意條款後就能安裝了
之後只要在xcode中 上面功能列Product - Destination - Show All Run Destinations選擇後 會出現有rosetta版本的模擬器 選擇該模擬器build & run即可

android studio

幾乎是沒有什麼問題 大多情形專案移植過去後僅需要 clean / rebuild project大部分就可以解決了

最多是Gradle路徑會錯誤而已 但是整體專案都不影響

homebrew

當初M1&M2版本安裝會有一點問題 但是現在都沒問題了 依照官網操作 安裝然後新增path就可以了

心得

從當初M1開始一堆東西會產生錯誤的情形到現在很順利的移植

更不用提當初一堆人不看好 到現在逐漸穩定 (看看某i的股價 不知道他們有沒有很後悔)

背後也是一堆人努力的付出 蘋果真的要好好感謝當初那群開發者們

感謝這些開發者的努力回饋與付出
